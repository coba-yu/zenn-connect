## はじめに

※この記事はClaude Codeが生成した内容をベースにしています。

## CLAUDE.md とは

CLAUDE.md は、Claude Code（Anthropic の公式 CLI ツール）に対して、プロジェクト固有またはグローバルな指示を与えるための設定ファイルです。このファイルを使うことで、Claude の振る舞いをカスタマイズし、チームやプロジェクトの開発フローに合わせた自動化を実現できます。

- [Claudeのメモリを管理する](https://docs.claude.com/ja/docs/claude-code/memory)
- [Claude Code: Best practices for agentic coding](https://www.anthropic.com/engineering/claude-code-best-practices)

## 記述内容の例

### Slack 通知の自動化

タスク完了時に Slack 通知を必須にする設定例：

```markdown
## You must follow these rules

### **Slack Notification (MANDATORY)**

After completing ALL tasks, you **MUST ALWAYS** send a completion notification to the Slack channel.

This is **NON-NEGOTIABLE** and must never be skipped.

- Notification content: Brief summary of the task and results
- Task completion definition: When all work requested by the user has been finished
```

### コミットメッセージの規約

Conventional Commits を強制する設定例：

```markdown
### Commit Message Specification

- All commit messages must follow the [Conventional Commits](https://www.conventionalcommits.org/) specification.
```

### その他の有用な記述例

**コードレビューの観点を指定**

```markdown
### Code Review Guidelines

When reviewing code, always check:
- Performance implications
- Security vulnerabilities
- Test coverage
- Documentation completeness
```

**テスト実行の義務化**

```markdown
### Testing Requirements

Before creating a pull request:
1. Run all unit tests
2. Run integration tests
3. Verify code coverage is above 80%
```

**ドキュメント更新の自動化**

```markdown
### Documentation Updates

When modifying public APIs:
- Update README.md with usage examples
- Update CHANGELOG.md with changes
- Add JSDoc/docstring comments
```

## ベストプラクティス

### 1. 明確で具体的な指示を書く

曖昧な指示ではなく、具体的なアクションを指定します。

```markdown
❌ "Be careful with commits"
✅ "Always run `npm test` before creating a commit"
```

### 2. 優先度を明示する

必須事項は **MANDATORY** や **MUST** などの強い表現を使います。

### 3. チーム全体で共有する

プロジェクトローカルの CLAUDE.md は Git で管理し、チーム全体で統一されたルールを適用します。

### 4. 定期的に見直す

プロジェクトの成長に合わせて、CLAUDE.md の内容も更新していきます。

## 実際の効果

CLAUDE.md を活用することで：

- **ヒューマンエラーの削減**: 必須タスク（テスト実行、通知送信など）の実行漏れを防止
- **開発フローの統一**: チーム全体で同じルールを適用
- **オンボーディングの効率化**: 新メンバーは CLAUDE.md を読むだけでプロジェクトのルールを理解できる
- **自動化の促進**: Claude が自動的にルールに従って作業を進める

## まとめ

CLAUDE.md は、Claude Code を使った開発フローを最適化する強力なツールです。プロジェクトの特性に合わせて適切に設定することで、開発効率と品質の両方を向上させることができます。

まずは小さなルールから始めて、徐々に拡張していくことをお勧めします。
